# Code of Conduct

Welcome to the Solar Network app project aka Solian (or you can call it's code Island)!

We're welcome for any contribution,
from bug reports to feature requests to code contributions.

## Getting Started

Before you continue, please read and agree to the Solsynth's CLA:
<https://solsynth.dev/legal/contributor-license>

Besides, we're not accepting pure AI contribution,
you at least need have the ability to code that language in order to review AI changes.

We won't enforcing any human edit percentage in your PR,
that will be inappropriate these days.

But once we found out that you're using AI without review and
proper testing to contribute to our projects.
We'll block you from any further contributions to ANY Solsynth projects
in order to save our members' time to review the AI slop's code.

Besides, a Pull Request MUST be linked to an issue, we need learn the context
about why you write these code, even a simple bug fix.

_For localization updates, you can directly go start the PR w/o linking an issue_

If you properly read and understand the content above,
feel free to fork the project and start making changes.

## Project Structure

The current repository you're visiting is the front-end project for the Solar Network project. It's built by Flutter and also manages all feature requests and issues reports in this repository.

The backend of the Solar Network is written in Go and is a microservices app. The code is stored separately in different repositories. They're linked in the README.MD, you can have a look and try to contribute if you want.

## Commit Messages

We're using the gitmoji to clarify the reason and changes of the commit.
To learn more about gitmoji, visit <https://gitmoji.dev>

All the commit message should follow `:[gitmoji]: <commit message>` syntax

## New Features

To contribute new features, please create an issue or mention the feature you want in our official development chat channel. You should discuss the feature with us and the community first. You shouldn't just create a Pull Request for the feature you want, it will not be merged.

## Bug Reports / Ask for help

Read the error message, check for the update (including pre-releases), and wiki before creating an issue. At the same time, be respectful and don't argue with our developers and contributors in the development chat or GitHub issue. Otherwise your issue may got deleted and your Solar Network Account may got a strike.

## Styles of Code

Before you create a Pull Request,
make sure your code has pass the `flutter analyze` check,
if there is any notes, fix as much as possible, if there is no way to fix,
do ignore comments to disable the analyze on that line explicitly.

When the code contains comments, use English.
We do not any other language of comments existing in the codebase.
It might confuse future contributors, cause the code hard to understand and maintenance.
